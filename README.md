# Student Survivor

A Flutter iOS-ready app wired to Supabase with a simple realtime todo list.

## Supabase Setup

Create a new Supabase project, then create the `todos` table with the fields
below. This demo uses the anon key, so keep policies open only for development.

Recommended table schema:

- `id` int8 primary key (identity)
- `title` text not null
- `is_done` boolean default false
- `created_at` timestamp with time zone default now()

Example SQL:

```sql
create table public.todos (
  id bigint generated by default as identity primary key,
  title text not null,
  is_done boolean not null default false,
  created_at timestamptz not null default now()
);

alter table public.todos enable row level security;

create policy "allow anon access" on public.todos
for all
to anon
using (true)
with check (true);
```

## Run

Pass your Supabase credentials as Dart defines:

```bash
flutter run \
  --dart-define=SUPABASE_URL=YOUR_URL \
  --dart-define=SUPABASE_ANON_KEY=YOUR_ANON_KEY
```

## iOS Bundle ID

The iOS bundle identifier is set to `com.example.studentSurvivor`.
